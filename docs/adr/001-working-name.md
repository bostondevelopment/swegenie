# ADR-001: Working name and domain candidates

**Status:** Accepted (working name only — final brand name is a Phase 4 decision, ADR-004)
**Date:** 2026-07-08

## Decision

Use **CareerGuru** as the internal working name (matches the repo). It is explicitly a
placeholder: "guru" reads career-coach/HR-flavored, which conflicts with the intended
credible-technical tone (levels.fyi / Pragmatic Engineer energy), and the .com is almost
certainly taken/parked.

## Domain candidates to evaluate in Phase 4

Criteria: memorable, self-describing or evocative of "map of engineering roles", pronounceable,
.com or credible alt-TLD available, handles available (GitHub, X, LinkedIn), no trademark
collisions.

Candidate directions (availability NOT yet checked — Phase 4 task):

| Direction | Examples |
|---|---|
| Map/landscape metaphor | roleatlas, engmap, rolemap.dev |
| Fit/match framing | rolefit.dev, engfit, nichefit |
| Archetype framing | archetype.dev, engarchetypes |
| Descriptive | whatkindofengineer.com, engineerroles.com |

## Consequences

- All Phase 0–3 artifacts use "CareerGuru (working name)" and avoid baking the name into
  user-facing copy, URLs, or package names where avoidable (app package is `app`).
- Phase 4 renames once: repo, domain, copy.
